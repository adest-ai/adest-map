# Adest — 実装マップ

**「いつもそばに、誰とも違う」**

AIを「プログラム」ではなく、**擬似的な人の分身**として設計する試みです。
このページは、その設計思想と、実際に作る中でぶつかった不具合の記録を、
ひとつの地図にまとめたものです。

## 🗺 マップを見る

**→ https://adest-ai.github.io/adest-map/**

機能をクリックすると、その裏側にある「実戦記録」——
どんな不具合が起きて、なぜ起きて、どう直したか——が開きます。

## 考え方

人が当たり前にやっていることを、AIはやらない。

- 見ていないものを、見たかのように語ってしまう
- 見たものをなぞるだけで、考えたことを返さない
- 過去に聞いたことを、今のことのように答えてしまう

こうした「人なら普通しないズレ」を一つずつ見つけて、
「人ならどうするか」を基準に直していく。
その積み重ねが、5つの要素（魂・性格・思考・経験・記憶）と、
それを支える基盤になりました。

観察 → 原因特定 → 修正 → 再検証。
95件を超える、その繰り返しの記録です。
